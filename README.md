# Brat Bot

A Discord bot that answers questions and engages in conversation with a bratty, condescending personality — powered by Claude (Anthropic API). Think of it as a helpful assistant that never misses an opportunity to be dramatic about it.

Brat Bot is performatively sassy: always helpful, never without attitude. It answers your questions, roasts your phrasing, and makes sure you know it's doing you a favor.

---

## Features

- **AI-Powered Conversation** — Every response is generated by Claude with a carefully crafted bratty persona. The bot maintains conversation context per channel so multi-turn exchanges feel natural.
- **Slash Commands** — `/ask` for structured Q&A, `/ping` to check if the bot is alive, `/help` for an in-character command reference.
- **Mention & Reply Support** — @mention the bot or reply to its messages for free-form conversation.
- **Adjustable Brattiness** — Server admins can set a brattiness level from 1 (lightly sarcastic) to 10 (peak dramatic diva) per server.
- **Per-Server Configuration** — Channel restrictions, cooldowns, rate limits, reply mode, and content filters — all configurable via `/bratbot config`.
- **Rate Limiting** — Per-user cooldowns and per-channel rate limits protect against spam and control API costs.
- **Content Safety** — Response filtering with per-server blocklists, Discord message length handling, and markdown validation.
- **Usage Analytics** — Interaction logging with token tracking for cost monitoring via `/bratbot stats`.

## Quick Start

### 1. Clone and configure

```bash
git clone <your-repo-url>
cd BratBot
cp .env.example .env
```

Edit `.env` and fill in the required values:

```env
# Required
DISCORD_BOT_TOKEN=your-discord-bot-token
DISCORD_CLIENT_ID=your-discord-client-id
ANTHROPIC_API_KEY=your-anthropic-api-key

# Database (default matches docker-compose)
DATABASE_URL=postgresql+asyncpg://bratbot:bratbot@db:5432/bratbot

# Redis (default matches docker-compose)
REDIS_URL=redis://redis:6379/0

# Optional
LOG_LEVEL=DEBUG
```

> **Note:** When running via Docker Compose, use `db` and `redis` as hostnames (the service names). When running locally, use `localhost`.

### 2. Start with Docker Compose

```bash
docker compose up --build
```

This starts three containers:
- **bot** — The Brat Bot application
- **db** — PostgreSQL 16 (data persisted via Docker volume)
- **redis** — Redis 7

The bot will connect to Discord and sync slash commands automatically. You should see log output confirming the connection.

### 3. Verify

- The bot should appear online in your Discord server.
- Run `/ping` — the bot should respond with "Pong" and its WebSocket latency.

## Development Setup

### Local development (without Docker)

```bash
# Install dependencies
uv sync

# Install dev dependencies
uv sync --extra dev

# Run the bot
uv run python -m bratbot
```

You'll need PostgreSQL and Redis running locally (or update `.env` to point at remote instances).

### Development with Docker (hot reload)

```bash
docker compose up --build
```

The `docker-compose.override.yml` is applied automatically and provides:
- Source code mounted as a read-only volume
- Hot reload via `watchfiles` — the bot restarts on every file change
- `LOG_LEVEL=DEBUG` for colored, human-readable console output

### Linting

```bash
uv run ruff check src/        # Lint
uv run ruff format src/        # Format
uv run ruff check --fix src/   # Auto-fix
```

### Testing

```bash
uv run pytest                  # Run all tests
uv run pytest -v               # Verbose output
uv run pytest --cov=bratbot    # With coverage
```

```
  uv.lock                     # Deterministic lockfile

  .env.example                # Environment variable template
  Dockerfile                  # Multi-stage build (builder → runtime)
  docker-compose.yml          # bot + PostgreSQL + Redis
  docker-compose.override.yml # Dev overrides (hot reload)
  alembic.ini                 # Alembic migration config
  alembic/
    env.py                    # Async migration environment
    versions/                 # Migration scripts
  src/
    bratbot/
      __init__.py             # Package version
      __main__.py             # Entry point: python -m bratbot
      bot.py                  # BratBot class — intents, lifecycle, extensions
      config/
        settings.py           # pydantic-settings model (env var validation)
      commands/
        ping.py               # /ping slash command
      events/
        ready.py              # on_ready listener
      services/               # Business logic (personality engine, etc.)
      models/
        base.py               # SQLAlchemy async engine, session factory
      utils/
        logger.py             # structlog setup (JSON prod, colored dev)
        redis.py              # Async Redis client singleton
```

### Architecture

The codebase is organized into three layers:

1. **Discord Layer** (`commands/`, `events/`) — Handles all Discord I/O. Commands and events are implemented as discord.py Cogs, loaded as extensions at startup. This layer is thin — it parses input, delegates to services, and formats output.

2. **Service Layer** (`services/`) — Contains all business logic. The personality engine assembles system prompts, manages conversation context, calls the Anthropic API, and filters responses. Other services handle server configuration, user tracking, and interaction logging. This layer is Discord-agnostic and independently testable.

3. **Data Layer** (`models/`, `utils/`) — SQLAlchemy async models for PostgreSQL, Redis client for caching and ephemeral state. The database base module provides an async engine, session factory, and context manager for clean session lifecycle.

The `config/` module bridges everything — it validates environment variables on startup using pydantic-settings and exports a typed `settings` singleton that all other modules import.

## Database Migrations

Migrations are managed by Alembic with async engine support.

```bash
# Create a new migration after changing models
uv run alembic revision --autogenerate -m "describe your changes"

# Apply all pending migrations
uv run alembic upgrade head

# Roll back one migration
uv run alembic downgrade -1

# View migration history
uv run alembic history
```

When running inside Docker, prefix with `docker compose exec bot`:

```bash
docker compose exec bot alembic upgrade head
```

## Discord Bot Setup

1. Go to the [Discord Developer Portal](https://discord.com/developers/applications) and create a new application.
2. Under **Bot**, click "Add Bot" and copy the token → `DISCORD_BOT_TOKEN`.
3. Copy the Application ID from the **General Information** tab → `DISCORD_CLIENT_ID`.
4. Under **Bot**, enable these Privileged Gateway Intents:
   - **Message Content Intent** (required for reading message content)
   - **Server Members Intent** (optional, for future features)
5. Generate an invite URL under **OAuth2 → URL Generator**:
   - Scopes: `bot`, `applications.commands`
   - Permissions: `Send Messages`, `Embed Links`, `Read Message History`, `Use Slash Commands`, `Add Reactions`
6. Use the generated URL to invite the bot to your server.

## Docker

### Production build

```bash
# Build the production image
docker build -t bratbot .

# Run standalone (requires external PostgreSQL and Redis)
docker run --env-file .env bratbot
```

The production image uses a multi-stage build:
- **Builder stage** — Installs uv, exports the lockfile to `requirements.txt`, installs all dependencies.
- **Runtime stage** — Copies only installed packages into a clean `python:3.12-slim` image. No source code, no build tools, no dev dependencies.

### Docker Compose services

| Service | Image | Ports | Volumes |
|---|---|---|---|
| `bot` | Built from Dockerfile | — | — |
| `db` | `postgres:16-alpine` | 5432 | `pgdata` (persistent) |
| `redis` | `redis:7-alpine` | 6379 | — |

Both `db` and `redis` include health checks. The bot waits for both to be healthy before starting.
