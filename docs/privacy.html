Privacy Policy

Effective Date: April 9, 2026

This policy explains what information BratBot collects when you interact with it on Discord, how that information is used, and how long it is kept.

1. What We Collect

BratBot collects the minimum information needed to operate. The table below lists every category of data, where it is stored, how long it is kept, and why.

Data Storage Retention Purpose
Discord User ID Redis Permanent Key for storing your preferences (verbosity, pronouns, age verification)
Verbosity preference (1–3) Redis Permanent Controls response length
Pronoun preference Redis Permanent Used by the language model to address you correctly
Age verification flag Redis Permanent Records that you accepted our Terms of Service
Channel ID Redis 60 seconds Per-channel rate limiting
Guild ID Redis 5 seconds Message deduplication on reconnect
Message content In-memory only Discarded after response Sent to a locally-hosted language model to generate a reply
Message IDs In-memory deque (max 1,000) Session only Prevents duplicate responses after Discord reconnections

Permanent means no automatic expiry — data remains in Redis until you request deletion (see Section 5).

2. How We Use Your Data

  • Message content is sent to a self-hosted language model (Ollama) running on our own servers. It is never forwarded to any third-party AI provider (e.g., OpenAI, Anthropic, Google).
  • Preference data is read from Redis at the start of each interaction to personalise the bot’s response.
  • We do not use your data for advertising, analytics, or any purpose beyond operating the bot.
  • We do not sell or share your data with any third party.

3. Data Retention

Message content is never written to disk and is discarded immediately after a response is generated. Preference data and your Discord User ID are stored in Redis with no expiry and persist until you request deletion. Temporary rate-limiting and deduplication data expires automatically within seconds.

4. Third Parties

  • Discord — messages pass through Discord’s infrastructure before reaching BratBot. Discord’s own Privacy Policy governs their handling of message data.
  • Ollama — self-hosted inference engine running on our servers. Message content stays within our infrastructure.
  • Redis / Upstash Redis — used for preference storage and rate limiting. On cloud deployments, Upstash Redis is used within our designated region.

5. Your Rights and Data Deletion

You may request deletion of all data associated with your Discord User ID at any time:

  1. Open an issue on our GitHub repository.
  2. Title it “Data Deletion Request” and include your Discord User ID.
  3. We will delete all Redis keys associated with your User ID and confirm in the issue.

If you are in the European Economic Area (EEA), United Kingdom, or California, you may also have rights to access, correct, or port your data under GDPR, UK GDPR, or CCPA/CPRA. Use the same GitHub issue process to make such a request.

6. Security

All Discord interaction webhooks are verified using Ed25519 signatures before processing. Message content is discarded immediately after a response is generated. Bot tokens and Redis credentials are stored as environment variables and are never logged or exposed.

7. Children

BratBot is not intended for users under the age of 18. Some bot personalities generate adult-themed content. We do not knowingly collect data from minors.

8. Changes to This Policy

If we make material changes to this policy we will update the effective date at the top of this page. Continued use of the bot after changes constitutes acceptance of the updated policy.

9. Contact

For questions about this policy or to exercise your rights, open an issue at github.com/z0rd0n88/BratBot/issues.
